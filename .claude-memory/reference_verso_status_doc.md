The living status doc for Verso is `where_we_are.md` at the root of the framing repo.

**Location:**
- Mac: `/Users/dpark/Manzano/framing/where_we_are.md`
- Windows: `C:\Users\YJ\manzano\framing\where_we_are.md`

**What it contains:** "Current focus" + "What's done" + "In flight" + "Queued" + "Decisions made" + "Open questions" + "Demo timeline" + "Notes for future-Drew."

**When to read it:**
- At the start of any Verso session, before answering "where are we" or "what should we do next."
- Before suggesting anything strategic — its decisions section overrides assumptions from memory snapshots.

**When to update it:**
- Every git commit to the framing repo — see [[framing-where-we-are-on-commit]].
- Tag each new bullet you add with `(Windows)` or `(Mac)` per [[framing-machine-tag]].

**Why this is in memory and not derived:** the doc lives in the repo so its content is always available via Read, but knowing to *check it first* is a habit that memory enforces.
